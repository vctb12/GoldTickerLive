# Contributing to Gold-Prices

Thank you for your interest in contributing! This guide will help you get up and running quickly.

## Project Overview

Gold-Prices is a bilingual (English / Arabic) gold price tracking platform for GCC and Arab markets. It is a **static, multi-page website** built with vanilla HTML, CSS, and ES6 JavaScript modules — no framework, no bundler at dev time. The production build is generated by Vite and deployed to GitHub Pages.

## Prerequisites

- **Node.js** — see `.nvmrc` for the required version (currently 22)
- **npm** — ships with Node.js

## Local Setup

```bash
# Clone the repository
git clone https://github.com/vctb12/Gold-Prices.git
cd Gold-Prices

# Install dependencies
npm install

# Start the dev server
npm run dev
```

The site will be available at `http://localhost:5173` (Vite default). You can also serve the static files directly:

```bash
python3 -m http.server 8080
```

Append `?debug=true` to any page URL to enable the debug panel.

## Code Standards

This project enforces code quality through three tools:

- **ESLint** — JavaScript linting (`eslint.config.mjs`)
- **Prettier** — Code formatting (`.prettierrc.json`)
- **Stylelint** — CSS linting (`.stylelintrc.json`)

Before submitting a pull request, ensure the quality check passes:

```bash
npm run quality
```

**Pre-commit hooks** are managed by [Husky](https://typicode.github.io/husky/) and run automatically when you commit. They execute ESLint, Prettier, and Stylelint on staged files via `lint-staged`.

## Testing

### Unit Tests

```bash
npm test
```

Tests use the Node.js built-in test runner (`node:test`) and assertion module (`node:assert`). Test files live in `tests/` and follow the `*.test.js` naming convention.

### Code Coverage

```bash
npm run test:coverage
```

### Playwright E2E Tests

End-to-end tests use Playwright. To run them locally:

```bash
npx playwright install --with-deps chromium
npm run build
npx playwright test --config=playwright.config.yml --project=chromium
```

E2E test files live in `tests/e2e/` and follow the `*.spec.js` naming convention.

## Pull Request Workflow

1. **Branch naming** — use a descriptive prefix:
   - `feat/` — new features
   - `fix/` — bug fixes
   - `chore/` — maintenance, refactors, dependency updates

2. **Before opening a PR:**
   - Run `npm run quality` and ensure it passes
   - Run `npm test` and ensure all tests pass
   - Write a clear PR description explaining what changed and why

3. **CI must be green** — the CI pipeline runs validation, build, tests, quality checks, coverage, and E2E tests. All checks must pass before merging.

## Project Structure

| Directory | Description |
|---|---|
| `src/` | Client-side JavaScript modules (config, lib, components, pages, tracker, utils, seo) |
| `styles/` | CSS files — `global.css` for design tokens, page-specific files in `styles/pages/` |
| `tests/` | Unit tests (`*.test.js`) and E2E tests (`tests/e2e/`) |
| `scripts/` | Build and utility scripts — `scripts/node/` for JS, `scripts/python/` for Python |
| `countries/` | Country, city, and market page templates |
| `config/` | Configuration files for external services |
| `server/` | Express API server (routes, repositories, services, lib) |
| `admin/` | Admin panel pages |
| `.github/workflows/` | CI/CD pipelines (CI, deploy, health checks, monitoring) |

## Reporting Bugs

Please use [GitHub Issues](https://github.com/vctb12/Gold-Prices/issues/new/choose) to report bugs. Use the **Bug Report** template and include:

- Steps to reproduce
- Expected vs. actual behavior
- Browser, OS, and the URL where the bug occurred
- Screenshots if applicable
