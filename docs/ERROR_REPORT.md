## Detailed Findings

### 1. Sitemap Contains Generated Page URLs

- **Severity**: Medium
- **File**: `sitemap.xml`
- **Issue**: The sitemap lists ~366 URLs. Of these, ~325 reference country/city/karat leaf pages
  (e.g. `/uae/dubai/gold-prices/`) that are generated by `build/generatePages.js`. These pages
  exist in the repo as committed HTML files but depend on the build step having been run.
- **Root cause**: Leaf pages are pre-generated and committed to the repo. The sitemap correctly
  references them. If leaf pages are ever deleted without regenerating, the sitemap would have
  broken links.
- **Fix**: Ensure `node build/generatePages.js` is run before any deployment that removes
  leaf pages. The deploy workflow currently copies these directories as-is.
- **Status**: Working as designed — leaf pages are committed to the repo.

### 2. Vite Build Fails on Node.js v24

- **Severity**: Medium (development only)
- **File**: `vite.config.js`, `package.json` (vite ^8.0.8)
- **Issue**: `npm run build` fails with `ERR_MODULE_NOT_FOUND` for Vite's internal chunk on
  Node.js v24.14.1. This is a Vite/esbuild compatibility issue with newer Node.js versions.
- **Root cause**: Vite 8.x may not fully support Node.js v24 yet.
- **Fix**: Use Node.js v22 LTS for builds, or wait for Vite to release a Node.js v24-compatible
  version.
- **Impact**: Production builds run in CI with a pinned Node.js version, so this only affects
  local development on bleeding-edge Node.js.

### 4. No Lazy Loading on Images

- **Severity**: Low
- **Issue**: No `loading="lazy"` attributes found on any `<img>` tags in the repo.
- **Root cause**: The site is mostly text-based with few images. The main images are icons, favicons,
  and OG images which don't benefit from lazy loading.
- **Fix**: Add `loading="lazy"` to any below-the-fold images if more images are added in the future.
- **Impact**: Negligible — the site has very few images.

### 6. Legacy Country Pages Coexist with Hierarchical Pages

- **Severity**: Low

- **Issue**: Two URL structures serve similar content for the same countries.

- **Fix**: Legacy pages have canonical tags. Consider adding 301 redirects from legacy to

- **Impact**: Potential duplicate content for SEO — mitigated by canonical tags.
